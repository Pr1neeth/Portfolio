LOG // NICE-01 · THESIS RESEARCH

Automata-Guided Imitation Learning for Long-Horizon Robotic Tasks

A neuro-symbolic imitation learning framework combining Probabilistic Deterministic Finite Automata (PDFA) with behavior cloning, built to improve long-horizon task execution from demonstrations on a dual-arm robot.

What I did

  • Collected teleoperated demonstrations on a dual-arm LeRobot setup, recording RGB observations and joint-position actions.
  • Used pretrained vision features and density-based clustering to discover task sub-goals directly from demonstrations.
  • Converted continuous robot trajectories into symbolic sequences using a PDFA-based task representation.
  • Combined high-level PDFA task planning with low-level behavior cloning policies.
  • Evaluated on long-horizon manipulation tasks including cube stacking and Three Card Monte.

Key results

  • Separated task structure/planning from low-level motor control in one framework.
  • Outperformed standard end-to-end behavior cloning on long-horizon execution.
  • Handled sequential dependencies, noisy demonstrations, and partial task failures more robustly.
  • Showed PDFAs can pinpoint exactly where task logic breaks when a robot completes only part of a sequence.

LOG // MARL-02 · MULTI-ROBOT RL

Bio-Inspired Multi-Robot Aggregation for Marine Waste Encapsulation

A multi-robot reinforcement learning system where autonomous surface robots learn, with no explicit communication, to surround drifting marine debris and form a protective ring around it.

What I did

  • Built a custom OceanTrashEnv grid-world simulation for the encapsulation task.
  • Formulated the task as a Multi-Agent Markov Decision Process with centralized observations.
  • Trained a centralized PPO policy to coordinate multiple robots.
  • Designed rewards for formation quality, fast convergence, and collision avoidance.
  • Tested against fixed debris and debris drifting under a time-varying tidal flow.

Key results

  • Policy converged from negative rewards to stable, positive performance.
  • Robots learned to form an approximate ring around debris with no explicit communication.
  • Static rollouts showed low formation error and very low collision rate.
  • Dynamic (drifting-debris) rollouts tracked debris but still need stronger reward shaping for robust encapsulation.

LOG // FR1-A · AIRFRAME & TVC

Design and Development of a Model Rocket with Active Stabilization

Designed and built God-Speed FR-1, a model rocket using thrust vector control to actively stabilize itself during flight.

What I did

  • Formulated and cast a custom F-class KNSU solid rocket motor (65% potassium nitrate / 35% sugar) — 0.07 kg dry mass, 0.085 kg propellant, 5.57 s burn time.
  • Designed a two-axis gimbal (inner = pitch, outer = yaw) actuated by SG90 servos, 3D printed and integrated into the motor mount.
  • Built the airframe from cardboard (recovery bay 0.3 m, avionics/propulsion bay 0.5 m, 78 mm OD) with a 3D-printed PLA ogive nose cone.
  • Fabricated a custom octagonal parachute recovery system with electric-match ejection and piston-based deployment.
  • Modeled the rocket in MATLAB/Simulink (3DOF), derived moment of inertia via a pendulum test, and tuned PID gains (P 0.65 / I 0.7 / D 0.2) for the gimbal loop.

Key results

≈1.0
thrust-to-weight ratio at launch
~12 N
max motor thrust
~12.1 N
vehicle weight
  • Ground testing validated full mechanical and avionics integration, with confirmed static margin.
  • Gimbal response, Kalman-filtered sensor fusion, and PID stabilization all worked as designed on the bench.
  • Flight attempt didn't achieve lift-off — the root cause was insufficient motor impulse, not a control failure, and it directly informed motor-sizing criteria for the next iteration.

LOG // FR1-B · FLIGHT COMPUTER

God-Speed Flight Avionics System (GSFTI)

A custom flight computer for rocket attitude sensing, data logging, thrust vector control, launch/burnout detection, and parachute deployment.

What I did

  • Designed a custom 2-layer PCB (Eagle, 100×60 mm, FR4) around an Arduino Nano (ATmega328P), MPU6050 IMU, BMP180 barometer, SD card module, pyro MOSFET channel, buzzer, and RGB status LED.
  • Implemented Kalman-filtered attitude estimation, cross-checked with a complementary filter, driving a real-time PID loop for the two gimbal servos.
  • Wrote the full flight-state machine in C++: launch detection (>2g Y-axis), burnout detection, apogee detection via barometric altitude, and a hard abort if pitch/roll error exceeds 45°.
  • Logged acceleration, gyro rate, fused angle, temperature, pressure, and altitude to SD card at high rate for post-flight analysis.

Key results

  • Built a fully self-contained flight computer with real-time 2-axis servo control and multi-sensor logging.
  • Verified every flight-state transition — launch, burnout, apogee, abort — through ground testing before the live attempt.
  • Successfully triggered recovery-deployment logic during ground validation.

LOG // UAV-06 · PROPULSION CONTROL

Thrust Controller Design for Propeller-Based UAVs

A thrust controller for propeller-based UAVs designed to hold motor thrust constant as air density shifts with altitude and pressure.

What I did

  • Studied the relationship between altitude, pressure, air density, RPM, and thrust using the ISA atmosphere model.
  • Modeled propeller thrust with blade element theory: T = CT × ρ × n² × D⁴.
  • Ran CFD analysis in ANSYS Fluent to study airflow, pressure, and velocity around the propeller.
  • Built a Simulink model to simulate thrust behavior under changing density conditions.

Key results

5.82 N
thrust at 6,000 RPM
13.5 N
thrust at 12,000 RPM
  • Measured thrust coefficient values directly from simulation data.
  • Built and validated a working thrust-control test bench.
  • Showed thrust can be held constant by increasing motor RPM as air density drops.
